# Milestone Schedule (Semester)

Instructions
- Update dates and deliverables to match your course timeline. Include owners for each milestone.

Milestones and deliverables

- Week 1 (YYYY-MM-DD to YYYY-MM-DD)
  - Deliverable: Finalized Problem Statement (root README)
  - Owner: [name/handle]
  - Acceptance criteria: Problem statement approved by supervisor; scope & success criteria defined.

- Week 2–3
  - Deliverable: Data inventory and access permissions
  - Owner: [name/handle]
  - Acceptance criteria: All primary data sources listed, licenses verified, sample dataset available.

- Week 4–5
  - Deliverable: Data ingestion & cleaning pipeline (first reproducible version)
  - Owner: [name/handle]
  - Acceptance criteria: Scripts produce cleaned sample dataset; data-dictionary completed.

- Week 6–7
  - Deliverable: Exploratory Data Analysis and initial visualizations
  - Owner: [name/handle]
  - Acceptance criteria: EDA notebook with documented figures and insights.

- Week 8–9
  - Deliverable: Modeling / deeper analysis (if applicable)
  - Owner: [name/handle]
  - Acceptance criteria: Models trained, evaluation results documented, baseline comparisons made.

- Week 10
  - Deliverable: Draft final report and presentation slides
  - Owner: [name/handle]
  - Acceptance criteria: Draft submitted for feedback.

- Week 11–12
  - Deliverable: Final deliverables: cleaned codebase, final report, polished visualizations, presentation
  - Owner: [name/handle]
  - Acceptance criteria: All code documented, reproducible environment provided, final slides ready.

Risk register (examples)
- Data access delayed — Mitigation: have fallback public datasets.
- Unexpected data quality issues — Mitigation: schedule additional cleaning week; document assumptions.

Communication cadence
- Weekly progress updates via GitHub issues; bi-weekly meeting with supervisor.
